Amigo

Soham Chatterjee

Amigo is an open-source data pre-processing and augmentation pipeline for Deep Learning workloads. It makes writing repetitive pre-processing, augmentation and data pipeline codes obsolete and at the same time reduces the bottlenecks that general data-pipelines have. Amigo means friend or sidekick and that is what it\u2019s job is: To handle the mundane data-preprocessing tasks (that ML engineers spend most time on), so that you can concentrate on the main task of training and improving your machine learning model.

eMark Wireless Multimeter

Projjal Gupta

eMark Wireless Multimeter is a device which can measure voltage and current values, similar to a regular multimeter, but it is wireless. It is supposed to be cheap and accessible to all, and the voltage and current values, along with power and other values can be directly read/calculated by a handheld device (Mobile Phones/Tablet/Laptops). The heart of the system will be a microcontroller and the wireless connection can be created using bluetooth/wifi. This needs to be accurate, cheap and should have features which support its build costs and quality.

Cone detection for driverless vehicle on race track using SSD-Mobilenet

Tenzin

This project is part of driverless project initiated by CamberRacing Team. Project that I am working on is an cone detection for driverless vehicle on race track using SSD-Mobilenet.\n\nSo i have been working on prototype with Raspberry pi installed and using donkey framework which has streering and throttle value prediction CNN model and i have successfully trained this basic model and achieved good performance in testing environment. After that I have planned to extend the feature of prototype to have object detection to detect cone and sementic segmantation to segment out the race track from surrounding to have region of interest, which will help in better prediction on steering angle and throttle value. So I have been working on SSD-Mobilenet for cone detection, since it is the most suitable for prototype running on Raspberry Pi. Still there are two things yet to complete, 1. Implementing image augmention and image transformation to increase the bounding box prediction accuracy. 2. Implementing mAP (mean Average Precision)
